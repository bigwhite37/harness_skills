# Review 范围漂移示例（负例）

这个示例展示了 `review` 发现 scope drift 后回退到 `build` 修正的合规处理路径。它是一个**负例**，演示流程如何捕获并纠正偏差。

## 来源系统体现

- `gstack`：固定阶段机完整执行，`review` 用对抗式审查视角主动发现范围漂移。
- `superpowers`：在活动 ticket 内做受控实现，回退后严格限制在 ticket 边界内重新实现。
- `harness engineering`：`review` gate 拦截范围漂移，要求回退 `build` 修正，`retro` 强制收尾。

## `reframe`

任务陈述:
为一个 API endpoint 增加请求参数校验，防止非法输入导致 500 错误。

成功标准:
- 非法参数返回 400 而不是 500
- 现有合法请求行为不变

非目标:
- 不重构 API 路由结构
- 不修改响应格式
- 不做性能优化

假设:
- 校验逻辑加在 handler 层即可，不需要中间件

未知项:
- 现有测试是否覆盖了非法输入场景

阻塞项:
- 无

完成定义:
ticket 被实现、review、verify 并以 `retro` 收尾。

## `plan`

已选方案:
在目标 handler 中增加参数校验，非法输入返回 400。

被拒绝的替代方案:
- 全局中间件校验（超出范围）
- 自定义校验框架（过度工程）

影响区域:
- 目标 handler 文件
- 一个定向测试文件

主要风险:
- 校验逻辑可能意外拒绝合法参数

验证契约:
- acceptance: 非法参数返回 400
  evidence: 定向测试结果
- acceptance: 合法请求行为不变
  evidence: 现有测试仍通过

回滚思路:
如果校验逻辑影响了合法请求，则回滚校验代码。

## `ticket`

Ticket 编号: T1
目标: 为目标 endpoint 增加参数校验
边界: 仅限目标 handler 和定向测试
输入: 已批准的 plan
预期产物: handler 中的校验逻辑和定向测试
验收点:
- 非法参数返回 400
- 合法请求不受影响
停止条件:
如果需要修改路由结构才能实现校验，返回 `plan`。

## `build`（第 1 轮）

当前 Ticket: T1

变更文件:
- 路径: handler 文件
  原因: 增加参数校验逻辑
- 路径: handler 文件（额外）
  原因: **顺手重构了响应格式统一化**
- 路径: 定向测试文件
  原因: 覆盖非法输入场景

新发现问题:
- 无（但实际引入了 scope drift）

## `review`（第 1 轮 — 失败）

状态: **失败**

问题:
- 严重度: 高
  问题: 响应格式统一化超出 ticket 边界，属于范围漂移
  修复: 回退到 `build`，移除响应格式变更，只保留参数校验

范围漂移:
- 响应格式统一化不在 ticket T1 的边界内，也不在 `plan` 的影响区域中

隐藏回退:
- 无

**判定: 回退到 `build`，移除超出范围的变更。**

## `build`（第 2 轮 — 修正后）

当前 Ticket: T1

变更文件:
- 路径: handler 文件
  原因: 仅保留参数校验逻辑，移除响应格式变更
- 路径: 定向测试文件
  原因: 覆盖非法输入场景

新发现问题:
- 无

## `review`（第 2 轮 — 通过）

状态: 通过

问题:
- 严重度: 无
  问题: 无
  修复: 无

范围漂移:
- 无（已在第 1 轮回退中修正）

隐藏回退:
- 无

## `verify`

验收点 -> 证据:
- 非法参数返回 400 -> 定向测试通过，断言状态码为 400
- 合法请求行为不变 -> 现有测试套件全部通过

已执行检查:
- 命令: 定向参数校验测试
  结果: 通过
- 命令: 现有测试套件
  结果: 通过

未验证项:
- 无

阻塞项:
- 无

剩余风险:
- 边界值参数（如极长字符串）未在本轮覆盖

结论:
`verified`

## `retro`

保留:
- `review` 的对抗式审查有效拦截了范围漂移

修改:
- 在 `build` 阶段增加自检：每个变更文件是否都能直接映射到活动 ticket

移除:
- "顺手改一下也没关系"的心理模式

流程后续:
- 考虑在 `build` 模板中增加"变更文件与 ticket 边界对照"检查项

## 交接

最终状态: `verified`
改动内容:
- 为目标 endpoint 增加了参数校验逻辑

已验证:
- 非法参数返回 400
- 合法请求行为不变

阻塞或未验证:
- 边界值参数覆盖未在本轮验证
