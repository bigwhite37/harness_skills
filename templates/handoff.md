# `handoff` 模板

## 目的

为下一位人类或智能体使用者总结最终状态。这是工作流结束后的产物，不是额外阶段。

## 输入

- 最终工作流状态
- 验证状态
- 剩余风险

## 输出

- 最终状态
- 改动内容
- 已验证内容
- 仍处于 `blocked` 或 `unverified` 的内容

## 通过条件

读者只看一眼就能判断本次运行是否结束于 `verified`、`blocked` 或 `reframe-needed`。

## 来源约束

> 来源约束见 references/boundary-rules.md

## 模板

```md
## 交接

最终状态:
改动内容:
- 

已验证:
- 

阻塞或未验证:
- 
```
