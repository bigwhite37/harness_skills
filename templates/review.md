# `review` 模板

## 目的

评估 `build` 输出是否正确、受边界约束且可维护。

## 输入

- 当前差异
- 活动 ticket
- `plan`

## 输出

- `gstack /review` 结论
- 发现的问题
- 严重度
- 通过或失败
- 修复方向

## 通过条件

已显式运行 `gstack /review`，且不存在未解决的高严重度问题，`review` 未发现范围漂移或隐藏回退。

## 来源约束

> 来源约束见 references/boundary-rules.md

## 模板

```md

- 调用结果:
- 关键发现:

- 严重度:
  问题:
  修复:

范围漂移:
- 无 | 详情

隐藏回退:
- 无 | 详情

持久化状态升级风险:
- 无 | 详情

复杂度审计:
- 是否超过 ticket 所需最小变更？详情:

测试契约对照:
- plan 约定的测试是否已落实？缺失项:
```
