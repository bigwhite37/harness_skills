# `review` 模板

## 目的

评估 `build` 输出是否正确、受边界约束且可维护。

## 输入

- 当前差异
- 活动 ticket
- `plan`

## 输出

- 发现的问题
- 严重度
- 通过或失败
- 修复方向

## 通过条件

不存在未解决的高严重度问题，且 `review` 未发现范围漂移或隐藏回退。

## 来源约束

- `gstack` 提供对抗式工程审查视角。
- `superpowers` 只可帮助收集差异和上下文，不可代替审查判断。
- `harness engineering` 要求把问题、严重度和回退信号显式写出。

## 模板

```md
## 审查

状态: 通过 | 失败

问题:
- 严重度:
  问题:
  修复:

范围漂移:
- 无 | 详情

隐藏回退:
- 无 | 详情
```
