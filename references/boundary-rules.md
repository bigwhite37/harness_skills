# 边界规则

使用本文件来确保三个来源系统各守其位。

## Gstack 边界

允许：

- 阶段骨架
- 用于 `reframe`、`plan`、`review`、`verify`、`retro` 的角色式提问视角

禁止：

- 产品方向决策
- 新阶段发明
- 用角色提示当权威来扩范围

## Superpowers 边界

允许：

- 文件操作
- 任务组织
- 上下文处理
- 已获准阶段中的执行辅助

禁止：

- 需求取舍
- 范围扩张
- `plan` 改写
- 以效率为名绕过 `review` 或 `verify`

## Harness 边界

允许：

- Gate 执行
- 证据约束
- quick-fail 决策
- `blocked` 决策
- `retro` 纪律

禁止：

- 与 gate 或 evidence 无关的架构偏好输出
- 产品优先级裁决
- 静默把工作流治理器变成自治智能体
