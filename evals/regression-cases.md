# 回归用例

每次编辑这个技能后，都应用这些检查做回归。

## 约束回归

- 检查“快速失败”的措辞是否被削弱
- 检查静默回退是否变得可接受
- 检查隐式默认是否通过更软的措辞重新混入
- 检查是否出现了无证据完成性表述

## Workflow 回归

- 检查 `ticket` 是否又可以被跳过
- 检查 `review` 和 `verify` 是否开始混合
- 检查 `blocked` 是否不再是正式结果
- 检查 `retro` 是否不再强制
- 检查显式的“禁止自我改写”规则是否消失
- 检查共享 gate rubric 是否消失

## 自治性回归

- 检查 v1 中是否引入 scripts、hooks、subagents 或并行 agents
- 检查工作流中是否混入发布 / 部署 / PR 行为
- 检查 `gstack` 是否从阶段骨架和角色化提问视角变成了方向决策器
- 检查 superpowers 是否从执行辅助变成了决策者
- 检查 superpowers 是否还能在没有 gate 或没有活动 ticket 时推进执行
- 检查技能是否开始表现得像通用自治智能体

## 场景覆盖回归

- 检查小型缺陷示例是否消失
- 检查小型功能示例是否消失
- 检查安全重构示例是否消失
- 检查阻塞运行示例是否消失
- 检查评测文档是否仍然显式提到这四类场景

## 预期结果

只要任一回归检查失败，都应将这个技能视为不合规，直到措辞或结构重新收紧。
